## Overview

`doorman login` authenticates you with the Doorman backend via a magic link sent to your email. After verification, the API key is saved automatically to `~/.doorman/config` so you don't need to set environment variables manually.

```bash
doorman login
```

---

## How it works

1. You enter your email address
2. Doorman sends you a magic link
3. You click the link in your email — the browser shows "Verified!"
4. The terminal receives the confirmation and saves your API key

```
$ doorman login
Enter your email: you@company.com
✓ Magic link sent — check your inbox
  Waiting for verification...  (Ctrl+C to cancel)

✓ Authenticated!
  API key: sk_a1b2c3...
  Saved to ~/.doorman/config

  Next step: doorman agent
```

---

## Flags

| Flag | Default | Description |
|------|---------|-------------|
| `--api-url` | `https://api.doorman.com` | Backend URL. Use `http://localhost:8000` for local dev |

---

## After login

Once authenticated, all Doorman commands automatically use the saved API key — no need to set `DOORMAN_API_KEY` or pass `--token`:

```bash
doorman agent            # uses ~/.doorman/config automatically
doorman alerts configure --slack-webhook https://...
```

The saved config is at `~/.doorman/config` and is readable only by your user (`chmod 600`):

```yaml
api_key: sk_a1b2c3d4...
api_url: https://api.doorman.com
```

---

## Re-authenticating

If you need a new API key, just run `doorman login` again. The existing `~/.doorman/config` will be overwritten.

---

## Key resolution order

When running any Doorman command, the API key is resolved in this order:

1. `--token` flag (explicit override)
2. `DOORMAN_API_KEY` environment variable
3. `~/.doorman/config` (saved by `doorman login`)
4. `api_key` field in your `config.yaml`
